DDR - Device Description Repository

What?

A collection of data sources that describe a device's capabilities according to an established vocabulary
How?

Java API proposed by the W3C Device Description Working Group
Who?

OpenDDR aimed to provide the reference community implementation; now part of ASF

Server-side client detection (OpenDDR)

Advantages

  1. bandwidth efficient
  2. no need for a smart client
  3. can target apps to a specific device

Disadvantages

  1. based on User-Agent headers
  2. detection can be affected by caches
  3. initial hit might be slow
  4. needs frequent data sources updates

Client-side features detection (BrowserMap)

Advantages

  1. detection based on actual browser features
  2. the client requests the best representation of a page
  3. detection not affected by caches
  4. easier to set up (at least the detection code)

Disadvantages

  1. needs a user agent that supports JavaScript
  2. might generate some additional server requests
  3. some browsers are still not standards compliant (:cough:BlackBerry:cough:IE 10)

  4. there are people who like to smear their laptop screens with their fingers :)
