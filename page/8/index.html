Where Has Java Gone From Lucid?

Apr 22nd, 2010

Apparently the Java packages coming from Sun (or Oracle for the matter) have been moved to the Partner repositories starting with the 10.04 release of Ubuntu, codenamed Lucid Lynx. The affected packages are:

  • ia32-sun-java6-bin

  • ia32-sun-java6-plugin

  • sun-java6-bin

  • sun-java6-demo

  • sun-java6-fonts

  • sun-java6-javadb

  • sun-java6-jdk

  • sun-java6-jre

  • sun-java6-plugin

  • sun-java6-source

To enable the use of the Partner repositories (which contain several more software packages), go to System » Administration » Software Sources and select the check-box in front of the partner repository line. The repository information will be uploaded and afterwards you can install the desired Java-related packages.

OpenX Ad Server: Beginner’s Guide

Apr 14th, 2010

UPDATE: You can find the review here.

Recently I’ve been contacted by Packt Publishing to review one of the books they published last month: OpenX Ad Server: Beginner’s Guide, by Murat Yilmaz. The book focuses on teaching web site administrators how to use and configure OpenX Ad Server, the largest publisher-side ad server in the world which has two major advantages: it’s an open-source product under the GNU GPL license and it’s free.

Addressed to enthusiasts with some basic IT knowledge (you should be able to know how to set up a web server, how to deploy web sites, etc.), this book should help you learn (according to the publishing company):

  • Learn the essentials of online advertising from the advertiser and publisher points of view

  • Install and utilize OpenX Ad Server effectively to make your business run faster

  • Get to know the usage of all the campaign and banner types

  • Manage ad campaigns and banners on multiple web sites from a centralized platform

  • Display ads according to their importance and alter the under-performing ads easily and swiftly

  • Learn the most effective way to work with web site ad zones

  • Maximize your profits by selling your ad space on your blogs and web sites effectively

  • Take full advantage of GeoTargeting for generating maximum revenue by showing ads according to visitor origin

  • Utilize the channels to show relevant ads based on the content of web sites

  • Integrate Google AdSense and Amazon ads with your web sites using OpenX

  • Convert OpenX Ad Server into a multi-user ad management platform for advertisers, publishers, and ad agencies

  • Track the success of any ad campaign, banner or web site zone using detailed statistics, and reports

The book will reach to me in eight to ten days. After I will read it thoroughly I’ll write a review on the blog to tell you if it worths its money or not. Until then, you can enjoy reading a sample chapter.

After I bought Riker , when it came to installing Ubuntu I had two options: either install the i386 version but with a PAE aware kernel, either go for the amd64 release. Considering the advantages and disadvantages of both of my options, I decided to go for the 64bit version (although the last time I did this I wasn’t quite happy – back in the 7.10 days). Anyway, Riker is extremely happy about it and I haven’t experienced any problems so far (I’m even running the 64-bit beta flash plugin which works great).

But let’s get back to the main point: Canon printers on 64bit Linux. While Canon does provide a Linux driver (both *.deb and *.rpm files), they are compiled to work only on a 32bit OS (which is like ½ Linux support). Therefore the only solution for the geeky misfortunate fellows like me is to compile their own drivers using Canon’s sources, but not following Canon’s instructions (which seem to be written by an unlucky lad who works at Microsoft but on UNIX services [this is a good joke if you are a geek]). While the process of compiling your own stuff is not very hard to accomplish (when you do have the proper instructions), trying to do this without knowing the exact dependencies is a major pain in the ass.

Fortunately, one guy on Ubuntu forums managed to create a proper set of steps to follow in order to do the job right. If you followed my blog, there’s a slight chance you might have noticed that I like to build BASH scripts to automate tedious Linux things. Since that post helped me to create the two *.deb files needed for the job, I decided to build a script just for the install part, using the packages I compiled and created on my 64bit machine. Also, because I really like geeking around, I made the script to install the correct drivers both on 32bit and 64bit (this is automatically handled by the script).

Geeks and Their Computers

Mar 16th, 2010

13 days ago I’ve bought another computer, thus reaching to number 3. My home network architecture is pretty simple:

As you might have noticed, my computers are named after Star Trek: The Next Generation characters. Let me describe them in their chronological order.

Picard is my former desktop which is now used primarily by my folks. Being an old timer, it’s the PC on which I’ve learned what’s Linux and on which I’ve developed several of my software projects from college. It still works okay, but comparing it with my latest acquisition, it’s slow and noisy.

Data was named like that because, at the time I bought it, it was the most powerful PC in the house, therefore resembling in a way with the famous android. I am still amazed by its capabilities when I compare the laptops available now in the same price range (it beats the crap out of them). My geekinees really emerged with the help of this machine, being capable of running Oracle DB, virtual machines, OC4J and many other power hungry appliances. It was the first machine on which, with the help of Mac4Lin, I began using the Mac OS X looks on an Ubuntu running computer (I never really quite liked the Human theme, nor any other default one). Since then all my machines share those looks (this will probably change with Ubuntu 10.04 which features an extremely nice looking theme, called Light).

Geordi is my beloved router, an Asus WL-500W draft N device which runs an alternate firmware that provides Samba and NFS access to the attached WD MyBook HDD, GIT (with gitosis), SVN, FTP, rTorrent (which accepts XML-RPC command through LIGHTTPD) and a dynamic DNS update client because my external IP is obtained using PPPoE. I’ve chosen Geordi as a name for the router because this device is responsible with the whole network and networking services, which is like The Engineering that La Forge led on the Enterprise. Also, the SSID for my wireless network is NCC-1701D (the code used for the USS Enterprise in the previous mentioned Star Trek series).

Geordi and Data have been bought together from my first salaries as an intern at Oracle (the place where I’ve began to master Linux).

Darwin is my iPod Touch 2nd generation. Its name is pretty self explanatory and it’s the only one not related to Star Trek, although I’m thinking to change it in the near future in order to keep my hostnames consistent (any suggestions?). For the ones who still don’t know what I’m talking about, check this out.

And here we are to Riker (Number One). Even if now this machine is the most powerful computing device I own, I still won’t change the name of my laptop. Since no other character name from TNG was worthy for this machine (maybe with the exception of Worf), Riker seemed to be the most appropriate one. Bought primarily for use at my diploma project (which is still a secret to the public), this computer outperforms Data and Picard together when it comes to processing power. Also I’ve opted for the i7 and the LGA 1366 socket due to their high performance architecture and their (currently) slow rate of becoming outdated.

Now tell me, how do you name your machines? Do you have a clear naming strategy like I do (FYI my naming strategy respects RFC 952, RFC 1123 and RFC 1178 [useful for non-geeks too])? :D
